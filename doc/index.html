Module connman_dbus

Get information about your network devices using Connman and DBus.

Requiring this module will return the Connman Manager singleton.

Example:

local Manager = require("connman_dbus"):init()
print(Manager.current_service.Type, Manager.current_service.Name)

Tables

Service
Connman service (exposed via Manager.services and Manager.current_service).

Fields:

  • dbus

    table containing the DBus information for the service:

    • path: the service's object path (string)
    • dest: "net.connman"
    • interface: "net.connman.Service"
    • bus: "system"
  • Name the SSID (like "MyWireless"), absent for ethernet and hidden Wifi
  • Type the service type, like "wifi" or "ethernet"
  • State one of "idle", "failure", "association", "configuration", "ready", "disconnect" or "online"
  • Strength WiFi signal strength between 0 and 100
  • Favorite bool, will be true if a cable is plugged in or the user selected and successfully connected to this service.
  • AutoConnect bool
  • Immutable bool
  • Ethernet

    table

    • Interface: like "wlo1", "eth0", etc.
    • Address: the MAC address
    • MTU: like 1500
    • Method: like "auto"
  • Security array of supported security, the elements can be: "none", "wep", "psk", "ieee8021x" or "wps"
  • Provider table
  • Domains array
    • Configuration table
  • Nameservers array
    • Configuration table
  • IPv4

    table

    • Netmask
    • Gateway
    • Address
    • Configuration

      table

      • Method
  • IPv6

    table

    • Netmask
    • Gateway
    • Address

      table

      • Method
  • Proxy table
    • Configuration table
  • Timeservers array
    • Configuration table
  • Error one of "out-of-range", "pin-missing", "dhcp-failed", "connect-failed", "login-failed", "auth-failed" or "invalid-key"
Manager

The Connman Manager singleton. This table is returned when the module is loaded with require. Call Manager:init() to initialize it.

E.g.

  Manager = require("connman_dbus"):init()

Fields:

  • State "offline", "idle", "ready" or "online"
  • OfflineMode bool
  • SessionMode bool (deprecated, always false)
  • techs

    table of technologies. Each key is the object path and each value is the table containing the tech's properties:

    • Powered: bool
    • Connected: bool
    • Name: str
    • Type: str ("ethernet", "wifi", etc.)
  • services array of services (sorted by Connman in order of preference)
  • current_service the currently active service
  • dbus table with DBus information
